# 练习记录 · 页面 PRD

> **页面类型**：管理后台类（PC 端）
> **所属模块**：考试中心
> **变更类型**：优化（已有页面，兼容新模式）
> **最后更新**：2026-07-01

---

## 1. 引言

### 1.1 页面概述

练习记录是管理后台「考试中心」模块下的**已有页面**，原先仅支持**专业题目练习（自由练习）**模式的记录统计。本次新增**模拟练习**模式后，页面需兼容两种练习模式的数据，将同一员工的自由练习记录与模拟练习记录**合并统计**展示在同一个列表中。

管理员可通过姓名、专业、部门进行筛选，支持导出和分页浏览。

### 1.2 变更说明

| 对比项 | 原页面（变更前） | 新页面（变更后） |
|--------|-----------------|-----------------|
| 数据来源 | 仅「专业题目练习」模式 | 「专业题目练习」+「模拟练习」两种模式 |
| 练习次数 | 专业练习次数 | 专业练习次数 + 模拟练习次数（合并） |
| 练习题数 | 专业练习题数 | 专业练习题数 + 模拟练习题数（合并） |
| 首次练习时间 | 专业练习的首次时间 | 两种模式中最早的时间 |
| 上次练习时间 | 专业练习的上次时间 | 两种模式中最晚的时间 |
| 其他 | — | 其余筛选、导出、分页功能保持不变 |

### 1.3 使用角色

| 角色 | 访问权限 |
|------|----------|
| 超级管理员 | ✓ 全量数据，可筛选 + 导出 |
| 安全管理员 | ✓ 按组织隔离，可筛选 + 导出 |
| 普通员工 | ✗（不可访问管理后台页面） |

### 1.4 前置条件

- 用户已登录管理后台
- 用户具有「考试中心 → 练习记录」菜单权限
- 系统中存在练习数据（专业练习或模拟练习已被员工参与）

## 4. 功能说明

### 4.1 筛选区

#### 4.1.1 筛选条件

| 条件 | 控件类型 | 占位/默认值 | 说明 |
|------|----------|-------------|------|
| 姓名 | 文本输入框 | 请输入 | 支持模糊搜索员工姓名 |
| 专业 | 下拉选择器 | 请选择专业 | 单选，选项来源于系统中已配置的专业列表 |
| 部门 | 下拉选择器 | 请选择 | 单选，选项来源于组织架构部门列表 |

#### 4.1.2 操作按钮

| 按钮 | 行为 |
|------|------|
| 查询 | 按当前筛选条件查询，刷新表格数据 |
| 重置 | 清空所有筛选条件恢复默认值 |

> **排列规则**：姓名 → 专业 → 部门 → 查询 → 重置，水平一行展示，不换行。

### 4.2 导出

| 按钮 | 位置 | 行为 |
|------|------|------|
| 导出 | 表格卡片顶部左侧 | 将当前筛选条件下的所有数据导出为 Excel 文件 |

> 导出范围为当前筛选条件下的**全部数据**（不受分页限制）。

### 4.3 数据表格

#### 4.3.1 数据合并规则

练习记录列表按**员工维度**聚合，将同一员工在两种练习模式下的数据合并展示：

| 合并字段 | 合并规则 |
|----------|----------|
| 练习次数 | 专业练习次数 + 模拟练习次数（累加） |
| 练习题数 | 专业练习题数 + 模拟练习题数（累加） |
| 首次练习时间 | 取两种模式中**最早**的练习时间 |
| 上次练习时间 | 取两种模式中**最晚**的练习时间 |

> **说明**：两种练习模式各自独立存储原始记录，列表展示时按员工维度实时聚合计算。员工只需参与过任一模式即出现在列表中。

#### 4.3.2 表格列

| 列名 | 说明 |
|------|------|
| 姓名 | 员工姓名；无姓名时显示 `-` |
| 部门 | 员工所属部门 |
| 所属专业 | 员工所属专业；无专业时显示 `-` |
| 练习次数 | 该员工累计参与练习的次数（两种模式合计） |
| 练习题数 | 该员工累计作答的题目总数（两种模式合计） |
| 首次练习时间 | 格式「年-月-日 时:分:秒」（两种模式中最早） |
| 上次练习时间 | 格式「年-月-日 时:分:秒」（两种模式中最晚） |
| 列设置 | 齿轮图标，点击可自定义显示列 |

#### 4.3.3 排序规则

按原有字段排序排列。

#### 4.3.4 行交互

鼠标悬停时行背景高亮，提供视觉反馈。

#### 4.3.5 列设置

表头最右列的齿轮图标，点击后弹出列显示/隐藏设置面板。

### 4.4 分页

| 元素 | 说明 |
|------|------|
| 总条数 | 显示「共 N 条」 |
| 每页条数 | 下拉选择：15条/页、20条/页、50条/页、100条/页 |
| 上一页/下一页 | `<` 和 `>` 箭头按钮，首页/末页时置灰禁用 |
| 页码按钮 | 数字按钮，当前页高亮 |
| 跳转 | 输入框 +「前往」+「页」，输入页码后回车跳转 |

## 5. 交互设计

### 5.1 页面加载

- 进入页面时自动加载练习记录列表数据（默认第 1 页，15 条/页，无筛选条件）
- 数据加载期间展示加载态

### 5.2 查询交互

- 点击「查询」：重新请求第 1 页数据，表格刷新
- 点击「重置」：清空姓名输入框、专业和部门下拉恢复默认「请选择」，同时自动触发查询刷新表格

### 5.3 导出交互

- 点击「导出」：按当前筛选条件导出全量数据为 Excel
- 导出成功后浏览器自动下载文件

### 5.4 分页交互

- 切换每页条数时重置为第 1 页
- 跳转输入框仅允许输入正整数，超出页码范围时自动纠正为最近的有效页码

## 6. 数据模型

### 6.1 练习记录数据结构

列表展示的是按员工聚合后的数据结构，底层分别存储两种模式的原始记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| 员工ID | 文本 | 员工唯一标识 |
| 姓名 | 文本 | 员工姓名 |
| 部门 | 文本 | 所属部门名称 |
| 所属专业 | 文本 | 所属专业名称 |
| 练习次数 | 数字 | 累计练习次数 = 专业练习次数 + 模拟练习次数 |
| 练习题数 | 数字 | 累计作答题目总数 = 专业练习题数 + 模拟练习题数 |
| 首次练习时间 | 日期时间 | 取专业首次时间与模拟首次时间中较早者 |
| 上次练习时间 | 日期时间 | 取专业上次时间与模拟上次时间中较晚者 |

### 6.2 底层原始记录（按模式分表存储）

#### 6.2.1 专业题目练习记录

| 字段 | 类型 | 说明 |
|------|------|------|
| 记录ID | 文本 | 唯一标识 |
| 员工ID | 文本 | 员工唯一标识 |
| 练习模式 | 枚举 | 「专业练习」 |
| 练习时间 | 日期时间 | 该次练习提交的时间 |
| 练习题数 | 数字 | 该次练习的题目数 |

#### 6.2.2 模拟练习记录

| 字段 | 类型 | 说明 |
|------|------|------|
| 记录ID | 文本 | 唯一标识 |
| 员工ID | 文本 | 员工唯一标识 |
| 练习模式 | 枚举 | 「模拟练习」 |
| 关联练习ID | 文本 | 关联的模拟练习配置ID |
| 练习时间 | 日期时间 | 该次练习提交的时间 |
| 练习题数 | 数字 | 该次练习的题目数 |

### 6.3 数据保留规则

- 练习记录与练习配置解耦存储，即使原练习配置被删除，记录仍然保留
- 即使练习人员被移出练习范围或从系统中删除，其已产生的练习记录仍然保留，不受人员变更影响
- 仅记录已完成提交的练习结果，未提交的练习不计入统计
- 专业练习和模拟练习的记录各自独立持久化，通过员工ID关联聚合

### 6.4 筛选条件数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | 姓名模糊搜索关键字 |
| 专业 | 文本 | 专业编号/名称 |
| 部门 | 文本 | 部门编号/名称 |
| 当前页 | 数字 | 当前页码（从 1 开始） |
| 每页条数 | 数字 | 每页条数（默认 15） |

## 7. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 无数据 | 表格区域展示空状态：「暂无数据」 |
| 接口请求失败 | 轻提示「加载失败，请稍后重试」，保留上次数据或展示空状态 |
| 导出失败 | 轻提示「导出失败，请稍后重试」 |
| 筛选无结果 | 表格展示空数据行，分页显示「共 0 条」 |
| 跳转页码超出范围 | 自动纠正为最近有效页码，不额外提示 |
