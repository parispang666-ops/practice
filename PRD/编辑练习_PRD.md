# 编辑练习 PRD（产品需求文档）

## 1. 引言

### 1.1 背景

编辑练习页面用于管理员修改草稿状态的模拟练习配置。页面需要回填已有数据，并根据当前部门、岗位和人员状态重新计算应练人数，帮助管理员在发布前处理离职或组织变更人员。

### 1.2 页面入口

| 来源 | 行为 |
|------|------|
| 练习列表草稿行点击「编辑」 | 跳转至 `编辑模拟练习.html` |
| index 导航点击「编辑练习」 | 打开编辑练习页面 |

### 1.3 使用角色

- **管理员**：可编辑草稿练习。
- **平台员工**：无此页面访问权限。

## 2. 页面布局

编辑练习为独立页面，包含 4 个表单分组。

```text
+-------------------------------------------------------+
|  ← 编辑模拟练习                                       |
+-------------------------------------------------------+
|  基本信息                                             |
|  选择试卷                                             |
|  练习题目数量                                         |
|  练习人员                                             |
|    部门（X个） 标签... 查看更多                       |
|    岗位（X个） 标签... 查看更多                       |
|    人员（X个） [一键清理离职人员] 标签... 查看更多     |
+-------------------------------------------------------+
|                 [返回] [保存]                         |
+-------------------------------------------------------+
```

## 3. 数据回填与人数计算

### 3.1 数据回填

进入编辑页面后，回填以下数据：

| 数据 | 说明 |
|------|------|
| 基本信息 | 练习名称、练习开放时间 |
| 试卷信息 | 已选择试卷及试卷结构 |
| 题目数量 | 题目总数、单选/多选/判断数量 |
| 练习对象 | 已选部门、岗位、人员 |
| 人员快照 | 创建练习时计算出的人员列表 |

### 3.2 人数重新计算

如果练习对象中包含部门或岗位，进入编辑页时需要根据当前组织和人员状态重新计算人数。

| 对象类型 | 计算规则 |
|----------|----------|
| 部门 | 根据当前部门下在职人员重新计算 |
| 岗位 | 根据当前岗位下在职人员重新计算 |
| 人员 | 直接回填已选人员，并标注已离职人员 |

统计人数时：

- 已离职人员仍可在已选人员中展示，但需标注「（离职）」。
- 部门、岗位范围内的当前在职人员用于计算应练人数。
- 同一人员同时命中部门、岗位或人员时，需要按人员 ID 去重。

## 4. 表单字段

| 字段 | 控件类型 | 必填 | 校验规则 | 交互说明 |
|------|----------|:----:|----------|----------|
| 练习名称 | 文本输入框 | 是 | ≤20 字 | 回填已有名称，可修改 |
| 练习开放时间 | 日期时间范围 | 是 | 结束时间 > 开始时间；开始时间 > 当前时间 | 回填已有时间，可修改 |
| 选择试卷 | 试卷选择弹窗 | 是 | 必选 | 回填已有试卷，可重新选择 |
| 练习题目总数 | 数字输入框 | 是 | >0 且 ≤ 试卷题目总数 | 回填已有数量 |
| 各题型数量 | 复选框 + 数字输入 | 是 | 至少选择一种题型；题型合计 = 总数；各题型不超过试卷上限 | 回填已有题型数量 |
| 练习人员 | 人员选择器 | 是 | 至少选择一个部门、岗位或人员 | 回填已有对象，支持继续添加或移除 |

## 5. 练习人员

### 5.1 已选内容展示

编辑页与新增页保持一致，按类型分组展示：

| 类型 | 展示标题 | 展示规则 |
|------|----------|----------|
| 部门 | 部门（X个） | 最多直接展示 10 项 |
| 岗位 | 岗位（X个） | 最多直接展示 10 项 |
| 人员 | 人员（X个） | 最多直接展示 10 项，离职人员标注「（离职）」 |

- 每个标签右侧展示删除图标，支持逐项移除。
- 去掉「查看已选」按钮。
- 去掉「全部移除」按钮。
- 当某类型已选数量超过 10 项时，在第 10 项后展示「查看更多X项」按钮。
- 点击「查看更多X项」打开对应类型的已选列表抽屉。

### 5.2 一键清理离职人员

「一键清理离职人员」按钮放在「人员（X个）」统计文字后面。

| 规则 | 说明 |
|------|------|
| 显示条件 | 已选人员中存在离职人员时展示 |
| 点击行为 | 移除已选人员中的离职人员 |
| 提示 | 「已清理X位离职人员」 |
| 提示位置 | 仅保留顶部 tip 提示 |
| 提示时长 | 自动消失，不持续展示 |

清理后：

- 人员标签中的离职人员被移除。
- 已选人员抽屉中的离职人员同步被移除。
- 人员统计数量更新。
- 页面只展示一个清理成功提示。

### 5.3 人员选择器

点击「添加人员」打开人员选择弹窗。

| 页签 | 查询条件 | 支持能力 |
|------|----------|----------|
| 部门 | 部门名称 | 支持分页、勾选多个部门 |
| 岗位 | 岗位名称 | 支持分页、勾选多个岗位 |
| 人员 | 姓名、部门名称 | 支持分页、勾选多个人员 |

分页规则：

- 部门、岗位、人员 mock 数据均支持超过 10 条。
- 默认每页 10 条。
- 切换分页不清空已选内容。
- 已选项在分页切换后仍保持勾选状态。

## 6. 查看更多抽屉

编辑页的查看更多与新增页完全统一，均为右侧抽屉展示。

| 抽屉 | 标题 | 查询条件 | 表格字段 | 操作 |
|------|------|----------|----------|------|
| 已选部门列表 | 已选部门列表 | 部门名称 | 序号、部门名称、操作 | 删除、批量删除 |
| 已选岗位列表 | 已选岗位列表 | 岗位名称 | 序号、岗位名称、操作 | 删除、批量删除 |
| 已选人员列表 | 已选人员列表 | 姓名、部门名称 | 序号、姓名、部门、操作 | 删除、批量删除、一键清理离职人员 |

### 6.1 已选人员抽屉

已选人员抽屉需要额外支持：

- 按姓名查询。
- 按部门名称查询。
- 已离职人员姓名后标注「（离职）」。
- 顶部展示「一键清理离职人员」按钮。
- 点击一键清理后提示「已清理X位离职人员」，提示自动消失。
- 清理后刷新抽屉列表和主表单人员标签。

### 6.2 抽屉通用规则

- 抽屉从右侧打开，宽度与新增页保持一致。
- 支持查询、重置。
- 支持勾选后批量删除。
- 未勾选时点击批量删除，提示「请选择要批量操作的内容」。
- 点击「返回」关闭抽屉并回到编辑表单。

## 7. 底部按钮

| 按钮 | 行为 |
|------|------|
| 返回 | 返回练习列表，不保存当前修改 |
| 保存 | 执行表单校验，校验通过后保存草稿 |

保存成功后提示：

> 练习已保存成功，您可以通过点击「发布」按钮完成发布

## 8. 表单校验

校验不通过时展示提示，不离开页面。

1. 练习名称必填。提示：「请输入练习名称」。
2. 请选择试卷。提示：「请选择试卷」。
3. 练习开放时间必须完整填写。提示：「请填写完整的练习开放时间」。
4. 结束时间必须大于开始时间。提示：「结束时间必须大于开始时间」。
5. 开始时间必须大于当前时间。提示：「开始时间必须大于当前时间」。
6. 题目总数必须大于 0。提示：「请输入有效的题目总数」。
7. 题目总数不能超过试卷题目总数。提示：「题目总数不能超过试卷题目总数（N 道）」。
8. 至少选择一种题目类型。提示：「请至少选择一种题目类型」。
9. 各题型数量之和必须等于题目总数。提示：「各题型数量之和需等于题目总数」。
10. 各题型数量不能超过试卷对应题型数量。提示：「不能超过试卷单选/多选/判断题数量（N 道）」。
11. 至少选择一个部门、岗位或人员。提示：「请至少添加一个人员或部门」。

## 9. 保存数据

| 字段 | 说明 |
|------|------|
| 练习名称 | 修改后的练习名称 |
| 试卷信息 | 当前选中的试卷 ID、名称、题目结构 |
| 题目数量 | 当前题目总数及各题型数量 |
| 开放时间 | 当前开始时间、结束时间 |
| 练习对象 | 当前已选部门、岗位、人员 |
| 人员快照 | 根据当前练习对象重新计算后的人员列表 |
| 发布状态 | 保存后仍保持草稿 |
