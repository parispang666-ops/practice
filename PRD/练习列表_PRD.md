# 练习列表 PRD（产品需求文档）

## 1. 引言

### 1.1 背景

管理员在 PC 后台通过「练习列表」统一查看、发布、删除和进入查看模拟练习配置。练习的新增与编辑已拆分为独立页面，本 PRD 仅覆盖练习列表页本身的功能。

### 1.2 使用角色

- **管理员**：可查看练习列表，进入新增练习、编辑练习、详情页，并对草稿练习执行发布、删除操作。
- **平台员工**：无此页面访问权限，仅在小程序端参与练习。

### 1.3 前置条件

- 用户已登录系统。
- 用户具有「练习设置」或「练习列表」菜单权限。
- 系统中已存在练习配置数据。

## 2. 页面范围

### 2.1 页面入口

| 页面 | 文件 | 说明 |
|------|------|------|
| 练习列表 | `练习设置.html` | PC 后台练习配置列表页 |
| 新增练习 | `新增模拟练习.html` | 从列表页点击「新增」进入 |
| 编辑练习 | `编辑模拟练习.html` | 从草稿行点击「编辑」进入 |

### 2.2 功能清单

| 模块 | 功能 | 功能描述 |
|------|------|----------|
| 列表区 | 练习列表 | 以表格形式展示所有模拟练习配置，按创建时间倒序排列 |
| 操作区 | 新增 | 点击进入独立的新增练习页面 |
| 操作区 | 详情 | 全部状态均可查看练习配置详情 |
| 操作区 | 发布 | 草稿状态可发布；发布前检测人员变更 |
| 操作区 | 编辑 | 仅草稿状态可编辑，点击进入独立的编辑练习页面 |
| 操作区 | 删除 | 仅草稿状态可删除 |

## 4. 需求说明

### 4.1 列表字段

| 字段 | 字段说明 | 排序 |
|------|----------|:----:|
| 练习名称 | 管理员自定义名称，≤20 字，单行展示 | 否 |
| 开始时间 | 练习开放开始时间，格式为年-月-日 时:分 | 否 |
| 结束时间 | 练习开放结束时间，格式为年-月-日 时:分 | 否 |
| 状态 | 草稿、未开始、进行中、已结束 | 否 |
| 创建人 | 创建该练习的管理员姓名 | 否 |
| 创建时间 | 创建时间，格式为年-月-日 时:分:秒 | 是，倒序 |
| 操作 | 根据状态展示可用操作 | 否 |

**排序规则**：按创建时间倒序，最新创建的练习排在前面。

**空状态**：列表无数据时显示「暂无数据」。

### 4.2 行操作

| 状态 | 操作按钮 | 说明 |
|------|----------|------|
| 草稿 | 发布、编辑、详情、删除 | 可发布、编辑、查看详情、删除 |
| 未开始 | 详情 | 可查看只读详情 |
| 进行中 | 详情 | 可查看只读详情 |
| 已结束 | 详情 | 可查看只读详情 |

### 4.3 新增

- 点击「新增」后跳转至 `新增模拟练习.html`。
- 新增页面的字段、校验和保存逻辑见 [新增练习 PRD](./新增练习_PRD.md)。

### 4.4 详情

- 全部状态均展示「详情」入口。
- 点击后进入只读详情视图，回填练习名称、试卷、题目数量、开放时间、练习人员等配置。
- 详情页展示的练习人员和共计人数以发布时人员快照为准；发布后发生离职、停用、转部门、转岗位，或有人员新进入部门/岗位范围，均不再修改详情页人数。
- 详情页全部字段禁用，仅提供「返回」。

### 4.5 编辑

- 仅草稿状态展示「编辑」入口。
- 点击后跳转至 `编辑模拟练习.html`。
- 编辑页面的字段、校验、人员变更和清理离职人员逻辑见 [编辑练习 PRD](./编辑练习_PRD.md)。

### 4.6 删除

- **显示条件**：仅草稿状态显示。
- **点击行为**：弹出确认弹窗，文案为「确定要删除该练习吗？此操作不可恢复。」
- **确认后**：从列表中移除该练习并刷新列表。
- **反馈**：提示「删除成功」，3 秒后自动消失。

## 5. 发布逻辑

### 5.1 发布前基础校验

发布仅对草稿状态开放，点击「发布」后依次执行以下校验：

1. 练习开放时间必须完整填写。失败提示：「请先编辑练习，填写完整的练习开放时间后再发布」。
2. 结束时间必须大于开始时间。失败提示：「结束时间必须大于开始时间，请先修改后再发布」。
3. 开始时间必须大于当前时间。失败提示：「开始时间必须大于当前时间，请先修改后再发布」。

### 5.2 人员变更检测

基础校验通过后，系统判断当前练习对象与创建练习时的人员快照是否存在变更。

| 变更类型 | 触发场景 | 展示规则 |
|----------|----------|----------|
| 移除 | 练习对象中的人员列表存在离职或停用人员 | 展示系统即将移除的人员列表和数量 |
| 移除 | 练习对象为部门范围，存在离职、停用或转出部门人员 | 展示系统即将移除的人员列表和数量 |
| 移除 | 练习对象为岗位范围，存在离职、停用或转出岗位人员 | 展示系统即将移除的人员列表和数量 |
| 新增 | 练习对象为部门范围，存在转入部门或启用人员 | 展示新进入练习对象的人员列表和数量 |
| 新增 | 练习对象为岗位范围，存在转入岗位或启用人员 | 展示新进入练习对象的人员列表和数量 |

> 页面文案统一使用「练习」，不使用「考试」。

### 5.3 有人员变更时

有人员变更时展示「发布」弹窗。

| 区域 | 内容 |
|------|------|
| 标题 | 发布 |
| 提示 | 【系统检测】与您创建练习时相比，人员列表已产生变更，变更信息如下： |
| 页签 | 全部、移除、新增，并展示对应数量 |
| 表格字段 | 序号、姓名、部门、操作类型 |
| 人员状态标注 | 已离职、已停用人员姓名后均标注「（离职）」 |
| 空状态 | 当前页签无数据时显示「暂无更新」 |
| 分页 | 支持 10 条/页分页 |
| 底部提示 | 练习发布后，新入职的人员不自动纳入练习中。 |
| 底部提示 | 练习发布后，离职、停用人员不影响练习流程，但会影响应练人数。 |
| 按钮 | 取消、发布 |

点击弹窗内「发布」后完成发布，关闭弹窗并提示「发布成功」。

### 5.4 无人员变更时

- 不展示发布弹窗。
- 直接完成发布并提示「发布成功」。

### 5.5 发布后状态

发布后将是否已发布设为已发布，练习状态由开放时间自动判定：

| 条件 | 状态 |
|------|------|
| 当前时间 < 开始时间 | 未开始 |
| 开始时间 ≤ 当前时间 ≤ 结束时间 | 进行中 |
| 当前时间 > 结束时间 | 已结束 |

## 6. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 列表数据为空 | 显示「暂无数据」 |
| 发布基础校验失败 | 展示对应提示，不进入发布流程 |
| 人员变更为空 | 不展示弹窗，直接发布成功 |
| 删除取消 | 关闭确认弹窗，不修改数据 |
| 前端模拟数据异常 | 保持当前页面，不清空已有列表 |
